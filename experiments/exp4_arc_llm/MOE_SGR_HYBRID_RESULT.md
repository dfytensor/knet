# SGR 学 MoE → 秩异构 MoE: 改进了 SGR, 但赢不了 MoE (诚实)

用户: "SGR 能不能学 MoE 改进一下"。诊断: SGR 的病=单条低秩路径饿死易 token;
MoE 的强=多专家专精不饿死。改进方案 = **秩异构 MoE**: SGR 的秩感知(专家不同秩) +
MoE 的多专家 top-k 路由。几个全秩专家(难 token 满血)+ 几个低秩专家(易 token 廉价专精)。

## 结果 (GLA 骨干, d256/L6, top-2, seq256, 1000步)

| 架构 | 总参 | val ppl | 说明 |
|------|------|---------|------|
| dense | 15.5M | 113.12 | 基线 |
| SGR-v3 原版(共享低秩+条件全秩) | ~17.8M | 106.6 | 单条低秩饿死 |
| HetMoE **2全秩+6低秩** | 19.4M | 107.62 | 学了 MoE 但低秩多 |
| HetMoE **4全秩+4低秩** | 21.8M | 103.10 | 全秩占比↑ |
| **MoE-uniform 8全秩** | 26.5M | **96.91** | 全满血专家 |

## 诚实裁定

### ✅ SGR 确实能学 MoE, 且明显改进
秩异构 MoE 比原版 SGR 强(HetMoE 4f4l ppl 103.10 < SGR-v3 106.6 < dense 113.12)。
**学 MoE 的多专家专精, 修掉了"单条低秩饿死"的病**——多专家让低秩专家各自专精, 不再所有易 token 挤一条弱路径。

### ❌ 但赢不了 MoE; 而且越学越变成 MoE
关键趋势: **全秩专家占比越高, ppl 越低, 单调收敛到 uniform MoE**:
2全秩(107.6) → 4全秩(103.1) → 8全秩(96.9, 即 MoE 本身)。
也就是说:
- **SGR 的"独特之处"(秩异构/低秩专家)恰恰是它掉点的原因**——低秩专家容量小。
- **"学 MoE"的过程 = 不断增加全秩专家 = 不断放弃 SGR 的秩特色 = 逐渐变成 MoE 本身。**
- 极限情况下"最好的 SGR-MoE"就是 MoE(全满血专家), SGR 的秩思想被完全淘汰。

### 参数效率也无优势
- MoE-uniform: 96.91 ppl / 26.5M
- HetMoE 4f4l: 103.10 ppl / 21.8M(省 18% 参, 但掉 6% ppl)
- HetMoE 2f6l: 107.62 ppl / 19.4M(省 27% 参, 但掉 11% ppl)
是 **ppl ↔ 参数** 的折中, 不是 Pareto 改进(多 22% 参换 6% ppl, MoE 划算)。

## 一句话(对"SGR 学 MoE")
> **SGR 能学 MoE, 学完(秩异构 MoE)确实比原版 SGR 强、也比 dense 强; 但它学得越多就越变成 MoE 本身,
> 而 SGR 真正独特的部分(低秩专家)正是它掉点的原因。** 换言之: **"SGR 学 MoE"的终点就是"放弃 SGR、用 MoE"**——
> 秩异构只在"想省参数、愿意掉点"时是个折中旋钮, 不是赢过 MoE 的新架构。
>
> 至此 SGR 的命运彻底清晰: 它的秩思想无论怎么改(单路/共享/异构/学 MoE), 都打不过 MoE 的"多满血专家专精"。
> **MoE 才是这条研究线里唯一真正有效的稀疏架构; SGR 的秩约束始终是个代价, 不是优势。**
