# CLR 对 MoE "决定性"? 实测: 不是, 反而轻微有害 (诚实反例)

用户判断: CLR 阀门原则对 MoE 架构 LLM "具决定性左右"。直接验证(真 sparse top-k MoE, GLA 骨干)。

## 结果 (GLA + FFN, seq256/d256/L6/1000步, E=8 专家)

| 配置 | 参数 | val ppl | 说明 |
|------|------|---------|------|
| dense | 15.5M | 113.12 | 基线 |
| **MoE fixed-k=2（从第0步）** | 26.5M | **96.91** | MoE 本身就赢 dense |
| MoE CLR-annealed（k: E=8 → 2） | 26.5M | 99.12 | **CLR 反而更差（+2.3%）** |

## 诚实裁定
### ❌ CLR 对 MoE 不"决定性", 反而轻微有害
- **MoE 不需要 CLR 就能赢 dense**（fixed-k=2 ppl 96.91 < dense 113.12）。稀疏 MoE 在此规模本就有效。
- **把 CLR 阀门原则用到 MoE（k 从 dense 退火到稀疏）反而更差**（99.12 vs 96.91）。
- 与 SGR 的结论**完全相反**：SGR 上 CLR 救命（+8.4%→+1.4%），MoE 上 CLR 帮倒忙。

### 为什么相反? (根因)
- **SGR 的失效模式**：稀疏路=低秩弱专家，从第0步把 token 路由过去会"饿死"→ CLR 退火（先 dense 拟合）能救。
- **MoE 没有这个问题**：每个专家都是**全容量 FFN**，top-2 路由从不"饿死"token（被选中的 2 个都是满血专家）。
  所以 MoE **不存在"需要先 dense 拟合"的病**，CLR 的"先拟合再压缩"对它是**无的放矢**——
  强行退火(k=8→2)反而给路由一个**移动靶**（早期学 8 专家分配，后期被迫重组成 2 专家），破坏一致性 → 掉点。
- **CLR 阀门原则是 SGR 弱路径失效模式的专用解药，不是通用 MoE 定理。**

## 修正后的判断 (对用户的"决定性"论)
> CLR 对 MoE **不具决定性**：MoE 本身（固定稀疏路由）就能赢 dense，CLR 退火反而轻微有害。
> CLR 阀门原则的有效性**完全取决于"稀疏路径是否饿死 token"**：
> - SGR（低秩弱专家）= 饿死 ⇒ CLR 救命；
> - MoE（全容量专家）= 不饿死 ⇒ CLR 无益甚至有害。
>
> 所以**"加 SGR 让 CLR 有效"是 SGR 特有的，不能外推到 MoE**。MoE 的稀疏是"在满血专家间选择"，
> 不需要 CLR 的"先 dense 再压缩"——它从一开始就稀疏得很好。

## 与全文的一致性
整条研究链再次印证: CLR/阀门原则是个**条件性工具**——
- grokking（强 gap）+ wd 旋钮 ⇒ 有效（§5）
- SGR（弱路径）+ frac 旋钮 ⇒ 有效（§11/16）
- 标准 LM（弱 gap）+ wd 旋钮 ⇒ 空转（§15）
- **MoE（满血专家）+ k 旋钮 ⇒ 有害（本节）**
没有任何"决定性/普适"的证据; 都是"看旋钮语义 + 看是否有可治之症"。
